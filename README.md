# Лабораторная работа: Каталог услуг (бинарный файл)

## Текст задачи

Разработать консольное приложение для работы с «базой данных» услуг, хранящейся в бинарном файле. Перечень полей (минимум 5): идентификатор, название, категория, цена, длительность в минутах. Приложение должно выполнять:

1. Чтение базы данных из бинарного файла.
2. Просмотр базы данных.
3. Удаление элементов (по ключу).
4. Добавление элементов.

В классе должны присутствовать свойства, конструкторы, перегруженный метод ToString(). Весь функционал приложения реализовать в виде методов вспомогательного класса с помощью LINQ-запросов. Предусмотреть обработку возможных ошибок при работе программы.

## Алгоритм решения

### Класс Service (файл Service.cs)

- Поля: `_id`, `_name`, `_category`, `_price`, `_durationMinutes`.
- Конструктор с параметрами.
- Свойства с валидацией:
  - Название и категория не пустые и не из пробелов.
  - Цена не отрицательная и ≤ 1 000 000 руб.
  - Длительность не отрицательная и ≤ 1440 мин.
- Статические методы `ValidateName`, `ValidateCategory`, `ValidatePrice`, `ValidateDuration` для единой проверки.
- Переопределённый `ToString()` для вывода.

### Класс ServiceManager (файл ServiceManager.cs)

- Статические методы:
  - `SaveToFile(List<Service>)` – запись списка в бинарный файл (BinaryWriter).
  - `LoadFromFile()` – чтение списка из бинарного файла (BinaryReader). Если файл не существует, возвращает пустой список. 
  - `GetAll()` – возвращает все услуги (вызов LoadFromFile).
  - `AddService(Service)` – добавляет новую услугу, автоматически назначает Id = максимальный существующий + 1 (LINQ Max).
  - `DeleteServiceById(int)` – удаляет услугу по Id, возвращает true если удаление выполнено, иначе false (LINQ FirstOrDefault).

### Класс Program (файл Program.cs)

- Бесконечный цикл с меню (пункты 1–4).
- Метод `ShowAllServices()` – вызывает `ServiceManager.GetAll()` и выводит результат.
- Метод `AddNewService()` – пошаговый ввод полей с немедленной валидацией через вызов `Service.ValidateXxx()`. Каждое поле переспрашивается до тех пор, пока не будет введено корректно.
- Метод `DeleteService()` – запрос Id, вызов `ServiceManager.DeleteServiceById()`, вывод сообщения.
- Обработка исключений:
  - `FormatException` – неверный числовой формат.
  - `ArgumentException` – нарушения валидации (сообщение из метода).
  - Общее исключение – для непредвиденных ошибок.
- При ошибке поле переспрашивается, остальные поля не сбрасываются.

### Работа с бинарным файлом

- Файл: `services.bin`.
- Формат записи:
  - Сначала количество записей (int).
  - Затем для каждой услуги: Id (int), Name (string), Category (string), Price (decimal), DurationMinutes (int).
- Чтение – в том же порядке с помощью BinaryReader.
